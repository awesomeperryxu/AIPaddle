# 设计说明：Copilot 自然语言生成 Workflow/Chatflow 与画布实时同步机制

> 2026-07-18 · 关联 ADR-005 · 落地任务 4.4.1 / 4.4.2 / 4.4.5 · 本文回答"自然语言自动生成 → 实时上屏 → 可视化调配"的实现机制

## 1. 单一数据契约（机制成立的根基）

- Workflow/Chatflow 唯一存在形式：`workflows.graph`（nodes + edges + 节点配置 JSON，节点类型 = PRD 2.5 Dify 风格 BlockEnum）
- ReactFlow 画布 = 渲染器；Copilot = 生成器；手动编辑 = 直接改同一份数据
- 禁止出现第二份"AI 专用"的流程表示格式

## 2. 生成链路（五步）

1. 画布右侧 Copilot 面板接收自然语言
2. Kimi 2.5 function calling 输出**图操作 patch 序列**：`add_node / update_node_config / connect / remove_node / remove_edge`
3. 服务端逐条执行：每条过 zod Schema + 图结构校验（无 Start/End、成环、孤立节点、Code 类节点【ADR-005 禁用】→ 拒绝）；校验错误自动喂回模型修正（生成-校验-修复循环，对用户无感）
4. 合法 patch 经 SSE 推送前端
5. 前端应用到画布 Zustand store → ReactFlow 增量渲染（节点逐个"生长"）

保存 = 整图落库 draft；发布走既有审核状态机（AI 不能触发发布）。

## 3. 实时性分两档实现（任务 4.4.5 内排序）

| 档 | 行为 | 说明 |
|----|------|------|
| MVP 档（先做） | 整图生成 → 校验 → 一次性替换画布 | 说完话 1-3 秒整图出现；实现最简 |
| 流式档（增强） | patch 协议 + SSE + 前端增量应用 | 画布逐节点生长 + 过程解说；体验质变 |

## 4. 双向协同规则（手动调配）

1. **单一状态源**：AI patch 与手动编辑写同一个 Zustand store、同一个 undo 栈（Cmd+Z 可撤销 AI 操作）
2. **AI 感知现状**：继续对话时，Copilot 工具先读画布当前图作上下文，在现状上出增量 patch，不整图覆盖用户手工修改
3. **谁改都是草稿**：发布永远人点

## 4.5 澄清交互面板（画布下方，PRD v1.09 场景 2）

- 生成中遇缺失配置 → 模型调用 `request_user_input` 产出问题清单（含 node_ref + field 定位）→ 渲染为画布下方可勾选面板
- 用户确认 → `submit_user_input` 回传 → 循环出 patch 回填节点 → 画布即时更新
- 节点"待配置"徽标 ↔ 面板问题双向定位；手改配置自动消解对应问题
- 意图路由跳转（PRD v1.09 场景 1）：全局对话识别创建意图 → 建 draft → 携会话上下文跳编辑页（右栏对话 + 左栏画布）

## 5. Dify 兼容

PRD 2.5 功能清单第 18 项 DSL 导入/导出（YAML）→ 做 Dify YAML ↔ 内部图 JSON 转换器即可迁入存量 Dify 工作流；切片 4 可选任务，无需新决策。

## 6. 验收挂钩

- 生成图必过 4.4.1 校验器（CP-04：含 Code 节点的图 422）
- S4-01/S4-03 覆盖保存恢复与执行；4.4.5 增补验收：手改后继续对话，AI 不覆盖手工修改（增量语义）
