# LLMOps SaaS 平台 - 租户内部管理模块 PRD

## 文档信息

**模块名称**: 租户内部管理（Tenant Internal Management）  
**文档版本**: 1.0  
**创建日期**: 2026-05-08  
**目标用户**: 企业租户管理员、HR、部门负责人

---

## 1. 产品定位

### 1.1 模块定位

本模块是 **租户自服务管理中心**，与 2.9 租户管理模块的区别：

| 对比项 | 2.9 租户管理模块 | 本模块（租户内部管理） |
|---|---|---|
| **使用者** | 平台运营方 | 企业租户管理员 |
| **管理对象** | 多个租户 | 单个租户内部 |
| **核心功能** | 租户开通、计费、模型治理 | 组织架构、人员、权限 |
| **权限级别** | 平台超级管理员 | 企业管理员 |
| **数据范围** | 跨租户 | 租户内隔离 |

### 1.2 核心价值

- **自主管理**: 企业无需依赖平台方，自主管理内部组织和人员
- **灵活配置**: 根据企业实际情况自定义部门、角色、权限
- **安全合规**: 完整的用户生命周期管理和审计日志
- **降本增效**: 精细化资源配额管理，控制成本

## 2. 核心功能模块

### 2.1 组织架构管理

#### 2.1.1 部门管理

**功能描述**: 企业内部部门的创建、编辑、删除和层级管理

**部门字段**:

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 部门名称 | String | ✓ | 如：研发部、市场部 |
| 部门编码 | String | ✓ | 唯一标识，如：DEV-001 |
| 上级部门 | Reference | - | 支持多级部门 |
| 部门负责人 | User | ✓ | 关联用户 |
| 部门描述 | Text | - | 职责说明 |
| 成本中心 | String | - | 财务编码 |
| 部门状态 | Enum | ✓ | 正常/冻结/已撤销 |
| 创建时间 | DateTime | ✓ | 自动生成 |

**核心功能**:

1. **部门树形结构**
   - 支持无限层级（建议不超过 5 级）
   - 拖拽调整部门层级
   - 批量移动部门
   - 部门合并/拆分

2. **部门操作**
   - 创建子部门
   - 编辑部门信息
   - 冻结部门（保留数据，禁止新增用户）
   - 撤销部门（需先转移人员）
   - 部门复制（快速创建相似部门）

3. **部门统计**
   - 部门人数（含子部门）
   - Token 消耗统计
   - Agent/Skill 使用情况
   - 部门活跃度

#### 2.1.2 部门资源配额

**配额类型**:

| 配额项 | 说明 | 默认值 | 可配置 |
|---|---|---|---|
| Token 月限额 | 部门每月可用 Token | 继承企业配额 | ✓ |
| 存储空间 | 知识库存储上限 | 10GB | ✓ |
| Agent 数量 | 可创建 Agent 数 | 10 | ✓ |
| Skill 数量 | 可创建 Skill 数 | 50 | ✓ |
| 并发调用数 | 同时调用上限 | 10 | ✓ |
| API 调用次数 | 每日 API 调用 | 10000 | ✓ |

**配额管理**:
- 配额分配（从企业总配额中分配）
- 配额监控（实时使用率）
- 配额告警（80%、90%、100% 阈值）
- 配额调整（管理员审批）
- 配额回收（部门撤销时）

### 2.2 人员管理

#### 2.2.1 用户信息管理

**用户字段**:

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 工号 | String | ✓ | 企业内唯一 |
| 姓名 | String | ✓ | 真实姓名 |
| 邮箱 | Email | ✓ | 登录账号 |
| 手机号 | Phone | - | 用于 MFA |
| 所属部门 | Reference | ✓ | 关联部门 |
| 直属上级 | User | - | 汇报关系 |
| 职位 | String | - | 如：高级工程师 |
| 入职日期 | Date | ✓ | 用于权限计算 |
| 用户状态 | Enum | ✓ | 正常/试用/离职/冻结 |
| 角色 | Array | ✓ | 可多角色 |
| 标签 | Array | - | 自定义标签 |

**用户状态流转**:

```
试用期 → 正式员工 → 离职
   ↓         ↓
  冻结 ←── 冻结
```

#### 2.2.2 用户创建

**创建方式**:

1. **单个创建**
   - 表单填写
   - 自动发送激活邮件
   - 设置初始密码（可选）

2. **批量导入**
   - Excel 模板下载
   - 数据校验（工号/邮箱唯一性）
   - 错误提示和修正
   - 导入预览
   - 批量激活

3. **企业微信同步**
   - OAuth 授权
   - 部门映射配置
   - 增量同步（每日自动）
   - 离职自动禁用
   - 同步日志

4. **飞书/钉钉同步**（未来支持）
   - 同企业微信逻辑

**Excel 导入模板**:

| 工号* | 姓名* | 邮箱* | 手机号 | 部门编码* | 职位 | 角色 |

| E001 | 张三 | zhang@company.com | 13800138000 | DEV-001 | 工程师 | 开发者 |

#### 2.2.3 用户生命周期管理

**入职流程**:
1. 创建用户账号
2. 分配部门和角色
3. 发送激活邮件
4. 用户首次登录设置密码
5. 完成入职培训（可选）

**在职管理**:
- 部门调动（保留历史记录）
- 角色变更（权限自动更新）
- 信息更新（姓名、手机、邮箱）
- 账号冻结（临时禁用）
- 密码重置

**离职流程**:
1. 标记为离职状态
2. 自动禁用登录
3. 回收所有权限
4. 转移负责的资源（Agent、Skill）
5. 归档用户数据
6. 保留审计日志（不可删除）

#### 2.3.3 数据权限

**数据权限级别**:

1. **全部数据** (All)

2. **本部门及下级部门** (Department+)
   - 查看本部门和所有下级部门数据

3. **仅本部门** (Department)
   - 仅查看本部门数据

   - 仅查看自己创建的数据
   - 适用于：普通用户

**数据权限应用**:

**分配方式**:

1. **直接分配**

2. **批量分配**

3. **自动分配**（未来）

**角色冲突处理**:
- 多角色权限取并集
- 明确禁止权限优先级最高
- 数据权限取最大范围

### 2.4 访问控制与安全

#### 2.4.1 单点登录 (SSO)

**支持协议**:
- SAML 2.0
- OAuth 2.0
- OIDC (OpenID Connect)
- CAS

**SSO 配置**:

| 配置项 | 说明 |
|---|---|
| IdP 地址 | 身份提供商 URL |
| Entity ID | 服务提供商标识 |
| 证书 | X.509 证书 |
| 属性映射 | 用户属性映射规则 |
| 登录 URL | SSO 登录入口 |
| 登出 URL | SSO 登出地址 |

**支持的 IdP**:
- Azure AD
- Okta
- Google Workspace
- 企业微信
- 飞书
- 钉钉
- 自建 IdP

#### 2.4.2 多因素认证 (MFA)

**MFA 方式**:

1. **短信验证码**
   - 登录时发送验证码
   - 有效期 5 分钟
   - 每日限制 10 次

2. **邮箱验证码**
   - 发送到注册邮箱
   - 有效期 10 分钟

3. **TOTP (Time-based OTP)**
   - Google Authenticator
   - Microsoft Authenticator
   - 支持备用码

4. **生物识别**（未来）
   - 指纹
   - 面部识别

**MFA 策略**:
- 企业级强制开启
- 角色级强制开启（如：管理员）
- 用户自主开启
- 信任设备（30 天免验证）
- 异地登录强制验证

#### 2.4.3 会话管理

**会话策略**:

| 配置项 | 默认值 | 可配置范围 |
|---|---|---|
| 会话超时 | 2 小时 | 30分钟 - 24小时 |
| 记住登录 | 7 天 | 1天 - 30天 |
| 最大并发会话 | 3 | 1 - 10 |
| 强制登出 | 支持 | - |
| 会话续期 | 自动 | 手动/自动 |

**会话监控**:
- 在线用户列表
- 会话详情（IP、设备、浏览器）
- 异常会话告警（异地、异常设备）
- 批量强制登出

#### 2.4.4 登录安全

**登录策略**:

1. **密码策略**
   - 最小长度：8 位
   - 复杂度：大小写+数字+特殊字符
   - 密码历史：不能重复最近 5 次
   - 密码有效期：90 天（可配置）
   - 首次登录强制修改

2. **登录限制**
   - 连续失败 5 次锁定账号
   - 锁定时长：30 分钟
   - IP 白名单（可选）
   - 时间段限制（可选）

3. **登录审计**
   - 登录成功/失败日志
   - 登录 IP 和设备
   - 登录时间
   - 异常登录告警

### 2.5 资源配额管理

#### 2.5.1 配额分配

**企业级配额**（从平台购买）:

| 资源类型 | 说明 | 计量单位 |
|---|---|---|
| Token 配额 | 每月可用 Token | Token |
| 存储空间 | 知识库存储 | GB |
| Agent 数量 | 可创建 Agent | 个 |
| Skill 数量 | 可创建 Skill | 个 |
| 用户数量 | 可创建用户 | 人 |
| API 调用 | 每日 API 调用 | 次 |
| 并发数 | 同时调用数 | 个 |

**部门级配额**（从企业配额分配）:
- 管理员分配给各部门
- 部门配额总和 ≤ 企业配额
- 支持动态调整
- 支持配额借用（临时）

**用户级配额**（从部门配额分配）:
- 部门管理员分配给用户
- 用户配额总和 ≤ 部门配额
- 个人限额保护

#### 2.5.2 配额监控

**监控维度**:

1. **实时监控**
   - 当前使用量
   - 剩余配额
   - 使用率
   - 预计耗尽时间

2. **趋势分析**
   - 每日使用趋势
   - 每周对比
   - 每月对比
   - 同比/环比

3. **异常检测**
   - 突增告警
   - 异常消耗
   - 配额即将耗尽
   - 超额使用

**告警策略**:

| 阈值 | 告警方式 | 接收人 |
|---|---|---|
| 80% | 邮件 | 部门管理员 |
| 90% | 邮件+站内信 | 部门管理员+企业管理员 |
| 100% | 邮件+站内信+短信 | 全部管理员 |
| 超额 | 自动限流 | - |

interface User {

## 5. 技术实现建议

### 5.1 权限控制

**推荐方案**: RBAC + ABAC 混合模式

- **RBAC**: 基于角色的访问控制（当前实现）
- **ABAC**: 基于属性的访问控制（未来扩展）

**实现方式**:
```typescript
// 权限检查
function hasPermission(
  user: User,
  module: string,
  action: string,
  resource?: any
): boolean {
  // 1. 检查角色权限
  const rolePermissions = user.roles.flatMap(r => r.permissions);
  const hasRolePermission = rolePermissions.some(
    p => p.module === module && p.actions.includes(action)
  );
  
  // 2. 检查数据权限
  if (resource) {
    const hasDataPermission = checkDataScope(user, resource);
    return hasRolePermission && hasDataPermission;
  }
  
  return hasRolePermission;
}
```

### 5.2 部门树性能优化

**问题**: 大型企业可能有数千个部门，树形结构渲染性能问题

**解决方案**:
1. 虚拟滚动（react-window）
2. 懒加载（按需加载子节点）
3. 缓存部门树结构
4. 使用 Materialized Path 或 Nested Set 模型

### 5.3 审计日志存储

**推荐方案**: 时序数据库 + 对象存储

- **热数据**（近 3 个月）: 时序数据库（InfluxDB、TimescaleDB）
- **温数据**（3-12 个月）: 关系数据库
- **冷数据**（1 年以上）: 对象存储（S3、OSS）
