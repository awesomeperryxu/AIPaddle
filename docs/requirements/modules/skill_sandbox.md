---

## 1. 模块概述

### 1.1 功能定位

Skill 是包含可执行代码的工具包。本模块负责其运行时:接收执行请求 → 从预热池分配沙箱 → 注入代码与参数 → 受限执行 → 回收结果 → 销毁沙箱。

与 Agent 运行时(`agent_runtime.md`)的关系:Agent 工具循环中所有"代码型工具"的执行请求都转发至本服务;纯 API 型工具直接走 MCP,不经过沙箱。

### 1.2 隔离技术

K8s Pod + RuntimeClass=gvisor(选型依据见《AI 平台基础架构设计》第 5 章),叠加 NetworkPolicy 与资源限额。

---

## 2. 沙箱池与生命周期

- **预热池**:常备 10 个待命沙箱(Python/Node 按 7:3 配比,可调);分配后异步补池
- **分配延迟**:池命中 P95 < 500ms;池空时冷启动兜底 < 3s
- **生命周期**:分配 → 注入代码 → 执行 → 回收产物 → **销毁 Pod**(绝不复用,防跨任务数据残留)

---

## 3. 关键规格(已确认)

### 3.1 资源限额(单沙箱)

| 项 | 限额 | 超限行为 |
|----|------|---------|
| CPU | 2 核 | 节流 |
| 内存 | 4 GB | OOM 终止 |
| 磁盘 | 512 MB(临时卷) | 写入失败 |
| 执行时长 | 5 分钟(硬超时) | 强制销毁,状态 timeout |

### 3.2 支持语言(一期)

- Python 3.12、Node 22 两个基础镜像
- **依赖预构建**:Skill 发布(过审)时,声明的依赖由 CI 构建进该 Skill 的专属镜像层;运行时不安装任何依赖
  - 收益:执行零等待、供应链安全(依赖经过漏洞扫描)
  - 代价:发布流程增加 1~2 分钟构建时间(已确认接受)

### 3.3 每人配额

- 并发沙箱:2 个
- 每日执行:100 次(超出可申请临时提额,租户管理员审批)

### 3.4 网络策略与联网信息获取(已确认)

**沙箱默认全禁出网,白名单仅两项:内部包镜像源(Nexus/Artifactory 代理 PyPI/npm)、LLM 网关。不开公网。**

联网需求的正确通道(能力不受损):

| 需求 | 通道 | 说明 |
|------|------|------|
| AI 检索互联网资料 | 平台统一 Web 检索 MCP 工具 | 部署在应用区,经受控代理出网;域名黑白名单 + 内容审计;检索结果进入模型上下文,不进沙箱 |
| 安装依赖 | 内部镜像源 | 已在白名单;且一期依赖为发布时预构建,运行时本就无需安装 |
| 调用外部 API(天气/汇率等) | 封装为 MCP 工具 | 出网发生在受控 MCP 服务上,凭证由 MCP 侧 Secret 保管,沙箱内代码只调 MCP |
| 调用企业内部系统(ERP 等) | 一律走 MCP | 与外部 API 同一原则(本次评审确认) |

**边界说明**:个人云端开发工作区(Coder)不是本模块的沙箱,网络策略单列且更宽——默认放行内部 Git、镜像源、LLM 网关,可按需申请追加官方文档类域名的只读访问。工作区内的高危执行(运行未经审查的 AI 生成脚本)转发到本模块的沙箱执行。

设计动机:沙箱断网的首要目的是**防数据外泄**——失控或恶意代码即使拿到了注入的数据,也没有把数据传出去的通道。

### 3.5 产物回传

- stdout / stderr:各截断 1 MB,超出部分丢弃并标记 truncated
- 输出文件:Skill 声明输出路径,执行结束后上传 MinIO,返回带时效的下载链接(默认 7 天)

## 4. API 定义

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/v1/sandbox/executions` | 提交执行(skill_id、版本、参数、关联 run_id) |
| GET | `/v1/sandbox/executions/{id}` | 查询状态与结果 |
| DELETE | `/v1/sandbox/executions/{id}` | 强制终止 |
| GET | `/v1/sandbox/quota` | 查询个人配额余量 |

错误码前缀:`SBX-`(如 `SBX-408-TIMEOUT`、`SBX-429-QUOTA`、`SBX-507-DISK`)

## 6. 性能与验收标准

1. 池命中时分配延迟 P95 < 500ms;冷启动 < 3s
2. 沙箱逃逸类系统调用 100% 被 gVisor 拦截(以公开逃逸测试集验证)
3. 超时/被杀任务 100% 完成资源回收,无僵尸 Pod(每小时对账巡检)
4. 出网尝试(白名单外)100% 被 NetworkPolicy 拒绝并产生告警事件
5. 沙箱销毁后临时卷数据不可恢复
