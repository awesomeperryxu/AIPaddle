## 2.5.1 模块概述

**功能描述**: 可视化 AI 工作流编排引擎，支持通过拖拽节点构建复杂的 AI 处理流程。

**两种模式**:

| 对比项 | Workflow（工作流） | Chatflow（对话流） |
|--------|------------------|------------------|
| 触发方式 | API 调用 / 定时 / Webhook | 用户对话消息 |
| 输出形式 | 结构化数据 / 文件 | 流式对话回复 |
| 特有节点 | End 节点 | Answer 节点 |
| 变量系统 | 系统变量 | 系统变量 + 对话变量 |
| 调试方式 | 单次运行 | 对话预览 |
| 典型场景 | 数据处理、文档生成 | 智能客服、问答助手 |

## 2.5.2 画布编辑器

### 核心布局

```
┌─────────────────────────────────────────────────────────┐
│  Header（顶部工具栏）                                      │
│  [标题编辑] [在线用户] [运行/历史] [环境变量] [版本历史]    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Canvas（画布区域）                                       │
│  ┌──────┐    ┌──────┐    ┌──────┐                       │
│  │Start │───▶│ LLM  │───▶│ End  │                       │
│  └──────┘    └──────┘    └──────┘                       │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  Operator（底部操作栏）                                   │
│  [撤销/重做] [变量检查] [缩放] [小地图]                    │
└─────────────────────────────────────────────────────────┘
```

### 画布交互

- **拖拽节点**: 从节点选择器拖入画布
- **连线**: 拖拽节点 Handle 连接到目标节点
- **多选**: 框选或 Shift+点击
- **缩放**: 滚轮缩放，Ctrl+0 重置
- **平移**: 空格键 + 拖拽，或中键拖拽
- **右键菜单**: 节点/边/画布各有独立右键菜单
- **注释模式**: 切换到 Comment 模式添加画布注释
- **协作光标**: 多人实时协作时显示他人光标

## 2.5.3 节点系统

### 节点分类

#### 触发节点（Trigger）

| 节点 | 说明 | 仅 Workflow |
|------|------|------------|
| Start | 工作流入口，定义输入变量 | - |
| TriggerSchedule | 定时触发 | ✅ |
| TriggerWebhook | Webhook 触发 | ✅ |
| TriggerPlugin | 插件触发 | ✅ |

#### AI 处理节点

| 节点 | 说明 |
|------|------|
| LLM | 大语言模型调用，支持多模型切换 |
| Agent | 智能体节点，支持多种 Agent 策略 |
| KnowledgeRetrieval | 知识库检索，支持多路召回 |
| QuestionClassifier | 问题分类，多分支路由 |

#### 逻辑控制节点

| 节点 | 说明 |
|------|------|
| IfElse | 条件分支，支持多 Case + ELSE |
| Iteration | 迭代节点，支持并行模式 |
| Loop | 循环节点，支持最大循环次数 |

#### 数据处理节点

| 节点 | 说明 |
|------|------|
| Code | 代码执行（Python/JavaScript） |
| TemplateTransform | Jinja2 模板转换 |
| VariableAssigner | 变量赋值 |
| ParameterExtractor | 参数提取（LLM 辅助） |
| ListFilter | 列表过滤/排序/去重 |
| DocExtractor | 文档内容提取 |

#### 外部集成节点

| 节点 | 说明 |
|------|------|
| HttpRequest | HTTP 请求，支持 GET/POST/PUT/DELETE |
| Tool | 工具调用（内置/自定义/MCP） |
| DataSource | 数据源连接 |
| KnowledgeBase | 知识库写入 |

#### 输出节点

| 节点 | 说明 | 模式 |
|------|------|------|
| End | 工作流结束，定义输出变量 | Workflow |
| Answer | 流式回复输出 | Chatflow |
| HumanInput | 人工介入，等待用户输入 | 两者 |

### 节点通用结构

```
┌─────────────────────────────────┐
│ ● [图标] 节点标题    [状态图标] │  ← 节点头部
├─────────────────────────────────┤
│ 节点描述（可选）                 │
├─────────────────────────────────┤
│ 节点内容摘要                     │  ← 节点体（各节点不同）
│ - 变量列表 / 模型选择 / 条件等   │
└─────────────────────────────────┘
  ○ 输入 Handle（左侧）
  ○ 输出 Handle（右侧）
```

**节点状态**:
- 默认：白色背景，灰色边框
- 选中：蓝色边框高亮
- 运行中：蓝色边框 + 旋转加载图标
- 成功：绿色勾选图标
- 失败：红色错误图标
- 异常：黄色警告图标
- 暂停：黄色暂停图标

### 容器节点（Iteration / Loop）

```
┌─────────────────────────────────────────────┐
│ ● [图标] Iteration 节点标题   [PARALLEL]    │
│ ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐ │
│   ┌──────────┐    ┌──────────┐             │
│   │ItStart   │───▶│  LLM     │             │
│   └──────────┘    └──────────┘             │
│ └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘ │
└─────────────────────────────────────────────┘
```

## 2.5.4 变量系统

### 变量类型

| 类型 | 说明 |
|------|------|
| String | 字符串 |
| Number | 数字 |
| Boolean | 布尔值 |
| Object | 对象（JSON） |
| Array[String] | 字符串数组 |
| Array[Number] | 数字数组 |
| Array[Object] | 对象数组 |
| File | 文件 |
| Array[File] | 文件数组 |

### 系统变量（sys.*）

| 变量 | 类型 | 说明 |
|------|------|------|
| sys.user_id | String | 当前用户 ID |
| sys.app_id | String | 应用 ID |
| sys.workflow_id | String | 工作流 ID |
| sys.workflow_run_id | String | 本次运行 ID |
| sys.timestamp | Number | 时间戳（仅 Workflow） |
| sys.dialogue_count | Number | 对话轮次（仅 Chatflow） |
| sys.conversation_id | String | 会话 ID（仅 Chatflow） |

### 对话变量（Chatflow 专有）

- 跨轮次持久化存储
- 支持 String / Number / Boolean / Object / Array 类型
- 通过 VariableAssigner 节点写入
- 在 Chat Variable Panel 中管理

### 环境变量（env.*）

- 存储 API Key、密钥等敏感信息
- 支持 String / Number / Secret 类型
- Secret 类型在 UI 中脱敏显示
- 通过 Env Panel 管理

### 全局变量

- 跨工作流共享的变量
- 通过 Global Variable Panel 管理

## 2.5.5 节点配置面板（右侧 Panel）

点击节点后，右侧弹出配置面板，包含：

### 通用配置

- **节点标题**: 可编辑
- **节点描述**: 可选填
- **错误处理**: 
  - 终止（Terminated）
  - 继续执行（Continue on Error）
  - 移除异常输出（Remove Abnormal Output）
- **重试配置**: 最大重试次数、重试间隔

### LLM 节点配置

- 模型选择（Provider + Model）
- 系统提示词（System Prompt）
- 用户提示词（User Prompt，支持变量插值）
- 上下文（Context，关联知识库检索结果）
- 输出变量（text / usage / finish_reason）
- 结构化输出（JSON Schema）
- 视觉能力（Vision，支持图片输入）
- 记忆（Memory，关联对话历史）

### IfElse 节点配置

- 多 Case 条件组（IF / ELIF / ELSE）
- 每个 Case 支持多条件（AND / OR）
- 条件操作符：等于、不等于、包含、大于、小于等
- 支持变量、字符串、数字比较

### Code 节点配置

- 语言选择（Python 3 / JavaScript ES6）
- 代码编辑器（Monaco Editor）
- 输入变量声明
- 输出变量声明
- 依赖包管理

### HTTP 节点配置

- 请求方法（GET / POST / PUT / DELETE / PATCH）
- URL（支持变量插值）
- Headers / Params / Body
- 认证方式（None / API Key / Bearer Token / Basic Auth）
- 超时设置
- 输出变量（body / status_code / headers）

### Iteration 节点配置

- 输入列表变量
- 并行模式开关（最大并行数 1-10）
- 错误响应模式（终止 / 继续）
- 输出变量

## 2.5.6 运行与调试

### 单次运行（Workflow）

1. 点击顶部「运行」按钮
2. 填写 Start 节点定义的输入变量
3. 点击「运行」执行
4. 右侧面板实时显示各节点执行状态
5. 执行完成后查看输出结果

### 对话预览（Chatflow）

1. 点击顶部「预览」按钮
2. 右侧弹出对话预览面板
3. 输入消息触发工作流
4. 实时查看节点执行状态和对话回复
5. 支持重置对话

### 节点单独运行

- 右键节点 → 「单独运行」
- 填写该节点所需的输入变量
- 查看该节点的输出结果

### 运行历史

- 顶部「历史」按钮查看历史运行记录
- 每条记录显示：运行时间、状态、耗时、Token 消耗
- 点击记录查看详细的节点执行轨迹
- 支持按状态筛选（成功 / 失败 / 运行中）

## 2.5.7 版本管理

### 版本历史面板

- 自动保存草稿（每次修改后）
- 手动创建命名版本
- 版本列表显示：版本号、创建时间、创建人、备注
- 支持筛选：全部 / 仅命名版本
- 操作：恢复、删除、导出 DSL、复制版本 ID

### 发布流程

1. 编辑工作流
2. 点击「发布」按钮
3. 填写版本备注（可选）
4. 确认发布
5. 发布后的版本可通过 API 调用

## 2.5.8 协作功能

- **在线用户**: 顶部显示当前在线编辑的用户头像列表
- **实时光标**: 显示其他用户的鼠标位置
- **节点占用**: 显示哪个用户正在编辑某个节点
- **注释系统**: 在画布上添加注释，支持回复和解决

  version: string
  created_at: string
  created_by: string
  marked_name?: string
  marked_comment?: string
}

## 2.5.11 API 接口

```
GET    /api/apps/:appId/workflows/draft          - 获取草稿
PUT    /api/apps/:appId/workflows/draft          - 保存草稿
POST   /api/apps/:appId/workflows/draft/run      - 运行草稿
GET    /api/apps/:appId/workflows/runs           - 获取运行历史
GET    /api/apps/:appId/workflows/runs/:runId    - 获取运行详情
POST   /api/apps/:appId/workflows/publish        - 发布工作流
GET    /api/apps/:appId/workflows/versions       - 获取版本列表
POST   /api/apps/:appId/workflows/versions/:id/restore - 恢复版本
```
