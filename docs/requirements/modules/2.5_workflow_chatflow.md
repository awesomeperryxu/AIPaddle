## 1. 模块概述

### 1.1 功能定位

Workflow 和 Chatflow 是平台的核心编排引擎，提供可视化的 AI 工作流设计能力：

- **Workflow**：面向批处理和 API 调用场景，以 End 节点输出结果，支持 Webhook/定时/插件触发
- **Chatflow**：面向对话场景，以 Answer 节点流式输出，支持多轮对话上下文和对话变量

### 1.2 技术架构

| 层次 | 技术选型 |
|------|---------|
| 画布引擎 | ReactFlow（节点/边/视口管理） |
| 状态管理 | Zustand（WorkflowStore + HooksStore） |
| 实时协作 | WebSocket + CRDT 冲突解决 |
| 草稿同步 | 自动保存 + 手动同步 |
| 历史记录 | 事件溯源（NodeAdd/Delete/Update/Move） |

## 2. 节点类型系统（30+ 种）

### 2.1 节点分类

#### 触发器节点（Trigger）
| 节点 | BlockEnum | 说明 |
|------|-----------|------|
| 开始 | `start` | 工作流入口，定义输入变量 |
| Webhook 触发 | `trigger-webhook` | 自动生成 Webhook URL |
| 定时触发 | `trigger-schedule` | Cron 表达式配置 |
| 插件触发 | `trigger-plugin` | 插件集成触发 |

#### AI 节点
| 节点 | BlockEnum | 说明 |
|------|-----------|------|
| LLM | `llm` | 大模型调用，支持提示词/记忆/视觉/推理格式/结构化输出 |
| Agent | `agent` | 智能体，支持策略选择、MCP 工具、记忆配置 |
| 问题分类 | `question-classifier` | 多分类路由 |
| 参数提取 | `parameter-extractor` | 从文本提取结构化参数 |

#### 逻辑控制节点
| 节点 | BlockEnum | 说明 |
|------|-----------|------|
| 条件分支 | `if-else` | IF/ELIF/ELSE 多分支，支持复合条件 |
| 迭代 | `iteration` | 数组迭代，支持并行（最大 100）、错误处理、展平输出 |
| 循环 | `loop` | 条件循环，支持循环变量、中断条件、最大次数 |
| 人工输入 | `human-input` | 暂停等待用户输入 |

#### 数据处理节点
| 节点 | BlockEnum | 说明 |
|------|-----------|------|
| 代码 | `code` | Python3/JavaScript 执行，自动解析函数签名 |
| 模板转换 | `template-transform` | Jinja2 模板渲染 |
| 变量赋值 | `variable-assigner` | 变量映射和赋值 |
| 列表操作 | `list-operator` | 列表过滤/排序/切片 |
| 文档提取 | `document-extractor` | 文档内容提取 |

#### 集成节点
| 节点 | BlockEnum | 说明 |
|------|-----------|------|
| HTTP 请求 | `http-request` | 支持 GET/POST/PUT/DELETE，CURL 导入，授权配置 |
| 工具 | `tool` | 动态表单，支持结构化输出 |
| 知识库检索 | `knowledge-retrieval` | 多知识库、多检索模式、元数据过滤、重排序 |
| 数据源 | `data-source` | 数据库连接 |
| 知识库索引 | `knowledge-base` | 知识库管理 |

#### 输出节点
| 节点 | BlockEnum | 说明 |
|------|-----------|------|
| 结束 | `end` | Workflow 出口，定义输出变量 |
| 直接回复 | `answer` | Chatflow 专属，支持变量插值，流式输出 |

## 3. 变量系统

### 3.1 变量类型（VarType）

```typescript
enum VarType {
  string = 'string',
  number = 'number',
  boolean = 'boolean',
  object = 'object',
  array = 'array',
  arrayString = 'array[string]',
  arrayNumber = 'array[number]',
  arrayObject = 'array[object]',
  file = 'file',
  arrayFile = 'array[file]',
}
```

### 3.2 变量作用域

| 作用域 | 前缀 | 说明 |
|--------|------|------|
| 系统变量 | `sys.*` | `sys.query`、`sys.user_id`、`sys.files` 等 |
| 节点输出 | `node_id.*` | 上游节点的输出变量 |
| 环境变量 | `env.*` | 工作流级别，支持加密存储 |
| 对话变量 | `conversation.*` | Chatflow 专属，跨轮次持久化 |
| 迭代变量 | `item` | 迭代节点内部当前元素 |
| 循环变量 | 自定义名称 | 循环节点内部变量 |

### 3.3 变量选择器（ValueSelector）

路径数组格式：`['node_id', 'output_key', 'nested_key']`

**UI 组件**：`VarReferencePicker`
- 节点列表 → 变量列表 两级选择
- 支持按 VarType 过滤
- 显示节点名称和变量路径

## 4. 节点配置面板详细规格

### 4.1 Start 节点面板

**字段**：
- 输入变量列表（可添加/删除/编辑）
  - 变量名、类型（String/Number/Array 等）、是否必填、默认值
- 内置变量（只读展示）：
  - Chatflow：`sys.query`（string）、`sys.files`（array[file]）
  - Workflow：`sys.files`（array[file]）

**交互**：
- 点击 "+" → 打开 `ConfigVarModal` 弹窗
- 删除变量 → 确认对话框（检查下游引用）

### 4.2 LLM 节点面板

**字段**：
1. **模型选择**：Provider + 模型名 + 参数（Temperature、Max Tokens 等）
2. **上下文**：变量选择器（引用上游文本变量）
3. **提示词**：System / User / Assistant 多段，支持变量插值 `{{var}}`
4. **Jinja2 变量**（可选）：动态变量列表
5. **记忆**（Chatflow）：开关 + 角色名称 + 消息数量
6. **视觉**：开关 + 分辨率（low/high/auto）
7. **推理格式**：tagged / xml
8. **结构化输出**：JSON Schema 编辑器

**交互**：
- 模型变更 → 自动验证并过滤不兼容的 completion_params
- 提示词支持 `@` 触发变量选择器

### 4.3 Code 节点面板

**字段**：
1. **输入变量**：变量名 + 变量引用选择器
2. **语言选择**：Python3 / JavaScript
3. **代码编辑器**：Monaco Editor，支持语法高亮
4. **代码生成器**：AI 辅助生成代码
5. **输出变量**：变量名 + 类型定义

**交互**：
- 代码变更 → 自动解析函数签名，同步输入/输出变量
- "同步函数签名" 按钮：手动触发同步

### 4.4 If-Else 节点面板

**字段**：
- IF 条件组（必须）
- ELIF 条件组（可多个，可添加/删除/排序）
- ELSE 分支（固定，无条件）

**每个条件组**：
- 多个子条件（AND/OR 逻辑运算符）
- 每个子条件：变量选择器 + 运算符 + 值

**支持的运算符**：
`contains`、`not contains`、`starts with`、`ends with`、`is`、`is not`、`empty`、`not empty`、`>`、`<`、`>=`、`<=`

### 4.5 HTTP Request 节点面板

**字段**：
1. **方法**：GET / POST / PUT / DELETE / PATCH
2. **URL**：支持变量插值
3. **授权**：None / Basic / Bearer Token / API Key（Header/Query）
4. **Headers**：键值对列表，支持变量插值
5. **Params**：键值对列表，支持变量插值
6. **Body**：none / form-data / x-www-form-urlencoded / raw（JSON/XML/text）
7. **SSL 验证**：开关
8. **超时**：秒级数字输入

**输出变量（固定）**：
- `body`（string）、`status_code`（number）、`headers`（object）、`files`（array[file]）

**交互**：
- CURL 导入按钮 → 解析 CURL 命令填充所有字段

### 4.6 Knowledge Retrieval 节点面板

**字段**：
1. **查询文本**：变量选择器（string 类型）
2. **查询附件**（可选）：变量选择器（file/array[file]）
3. **知识库列表**：多选，每个知识库可配置检索参数
4. **检索模式**：单一 / 多路 / 混合
5. **重排序模型**：模型选择器
6. **元数据过滤**：过滤模式（AND/OR）+ 条件列表

**输出变量（固定）**：
- `result`（array[object]）：含 content、title、url、icon、metadata、files

### 4.7 Iteration 节点面板

**字段**：
1. **输入数组**：变量选择器（array 类型）
2. **输出数组**：变量选择器
3. **并行模式**：开关 + 最大并行数（1-100，滑块+输入框）
4. **错误处理**：终止 / 继续执行 / 移除异常输出
5. **展平输出**：开关（用于嵌套迭代）

### 4.8 Loop 节点面板

**字段**：
1. **循环变量**：变量列表（名称 + 初始值 + 类型）
2. **中断条件**：条件编辑器（同 If-Else 条件格式）
3. **逻辑运算符**：AND / OR
4. **最大循环次数**：数字输入（防止无限循环）

### 4.9 Agent 节点面板

**字段**：
1. **策略**：策略下拉选择（Function Calling / ReAct 等）
2. **策略参数**：动态表单（基于策略 schema）
3. **输出 Schema**：自定义输出字段定义
4. **记忆**（Chatflow）：开关 + 历史消息数

**输出变量**：text、usage、files、json + 自定义字段

enum ControlMode {
  Pointer = 'pointer',  // 默认：选择、拖拽、编辑节点

## 6. 顶部 Header 规格

### 6.1 三种状态

| 状态 | 触发条件 | 显示内容 |
|------|---------|---------|
| Normal | 默认 | 标题编辑 + 运行/发布按钮 |

5. 发布按钮（下拉：发布 / 发布为 API / 发布为 Webhook）

### 6.3 运行模式

- **测试运行**：弹出参数输入面板，手动填写 Start 节点变量，单次执行
- **触发器运行**：Webhook / 定时 / 插件触发器配置
- **单节点运行**：右键节点 → 从此节点运行

## 9. Chatflow 专属功能

### 9.1 Answer 节点

- 富文本编辑器（支持 Markdown）
- 变量插值：`{{node_id.output_key}}`
- 文件变量引用
- 流式输出到对话界面

### 9.2 对话变量（Conversation Variables）

- 跨轮次持久化的变量
- 支持 string、number、boolean、object、array 类型
- 在 Variable Assigner 节点中赋值
- 在任意节点中引用（前缀 `conversation.*`）

### 9.3 记忆配置（Memory）

LLM 和 Agent 节点在 Chatflow 中额外支持：
- 记忆开关
- 角色名称（AI 角色名 / 用户角色名）
- 历史消息数量

### 9.4 Chatflow vs Workflow 差异对照

| 特性 | Workflow | Chatflow |
|------|----------|----------|
| 出口节点 | End | Answer |
| 内置变量 | `sys.files` | `sys.query`、`sys.files` |
| 对话变量 | 不支持 | 支持 |
| LLM 记忆 | 可选 | 支持 |
| Agent 记忆 | 不支持 | 支持 |
| 调试面板 | 运行结果面板 | 对话预览面板 |

## 12. DSL 导入/导出

### 12.1 导出格式（YAML）

```yaml
app:
  name: 工作流名称
  description: 描述
  mode: workflow | advanced-chat
workflow:
  nodes: [...]
  edges: [...]
  environment_variables: [...]
  conversation_variables: [...]  # Chatflow only
```

### 12.2 导入流程

## 16. 补充节点配置面板规格

### 16.1 Answer 节点面板（Chatflow 专属）

**字段**：
- 富文本编辑器（支持 Markdown）
- 变量插值：输入 `{{` 触发变量选择器
- 文件变量引用支持

**交互**：实时编辑，变量以彩色 chip 显示

### 16.2 Question Classifier 节点面板

**字段**：
1. **查询变量**：变量选择器（string 类型）
2. **模型选择**：Provider + 模型名
3. **分类列表**：可添加/删除/排序的分类项，每项含名称和描述
4. **指令**（可选）：自定义分类指令

**输出**：每个分类对应一个输出端口

### 16.3 Parameter Extractor 节点面板

**字段**：
1. **输入文本**：变量选择器
2. **模型选择**：Provider + 模型名
3. **参数列表**：参数名 + 类型 + 描述 + 是否必填
4. **指令**（可选）：提取指令

**输出变量**：每个参数对应一个输出变量

### 16.4 Template Transform 节点面板

**字段**：
1. **输入变量**：变量列表（名称 + 变量引用）
2. **模板编辑器**：Jinja2 语法，支持 `{{ variable }}` 插值

**输出变量**：`output`（string）

### 16.5 Variable Assigner 节点面板

**字段**：
- 赋值列表：目标变量（对话变量/环境变量）→ 源变量（变量选择器）
- 支持添加/删除赋值规则

**用途**：主要用于更新 Chatflow 的对话变量

### 16.6 List Operator 节点面板

**字段**：
1. **输入列表**：变量选择器（array 类型）
2. **操作类型**：过滤 / 排序 / 切片 / 去重
3. **操作参数**：根据操作类型动态显示
   - 过滤：条件编辑器
   - 排序：字段 + 升序/降序
   - 切片：起始索引 + 结束索引

**输出变量**：`result`（array）

### 16.7 Document Extractor 节点面板

**字段**：
1. **文件变量**：变量选择器（file/array[file] 类型）
2. **提取模式**：文本 / 表格 / 图片

**输出变量**：`text`（string）

### 16.8 Human Input 节点面板

**字段**：
1. **提示文本**：显示给用户的提示信息（支持变量插值）
2. **超时设置**：等待时间（秒），0 表示无限等待
3. **输入变量定义**：用户需要填写的变量列表

**交互**：工作流执行到此节点时暂停，等待用户通过 API 提交输入

### 16.9 Trigger-Webhook 节点面板

**字段**：
1. **Webhook URL**（只读，自动生成）：复制按钮
2. **请求方法**：GET / POST（只读）
3. **输出变量**（只读）：
   - `body`（object）：请求体
   - `headers`（object）：请求头
   - `query`（object）：查询参数

### 16.10 Trigger-Schedule 节点面板

**字段**：
1. **Cron 表达式**：输入框 + 可视化选择器
2. **时区**：时区选择器
3. **下次执行时间**（只读，自动计算）

### 16.11 Tool 节点面板

**字段**：
1. **工具输入参数**：动态表单（基于工具 schema）
   - 每个参数：标签 + 输入框/选择器/变量引用
2. **工具设置**（可折叠）：工具特定的配置项

**输出变量**：
- `text`（string）：文本输出
- `files`（array[file]）：文件输出
- `json`（array[object]）：JSON 输出
- 自定义输出字段（由工具 schema 定义）

### 16.12 End 节点面板

**字段**：
- 输出变量列表：变量名 + 变量引用选择器
- 支持添加/删除输出变量

**用途**：定义 Workflow 的最终输出，供 API 调用者获取

## 17. 补充功能（基于源码深度核查）

### 17.1 便签节点（Note Node）

**功能定位**：非执行节点，用于在画布上添加文字注释和说明。

**特性**：
- 支持富文本编辑（NoteEditor）
- 支持 8 种主题颜色（黄色/蓝色/绿色/紫色/粉色/橙色/灰色/白色）
- 支持作者信息显示（showAuthor）
- 支持节点大小调整（NodeResizer）
- 支持复制/删除/重复操作
- 工具栏功能：加粗/斜体/下划线/链接/列表

**UI 规格**：
- 最小宽度：240px
- 最小高度：120px
- 可自由调整大小
- 左上角显示主题颜色标签

### 17.2 对齐辅助线（Help Line）

**功能**：节点拖拽时自动显示水平/垂直对齐参考线。

**特性**：
- 支持缩放感知（zoom-aware）
- 实时计算节点对齐位置
- 虚线样式显示
- 自动隐藏

### 17.3 节点操作菜单（Node Actions Menu）

**触发方式**：节点悬停时显示下拉菜单按钮。

**菜单项**：
- 帮助链接（showHelpLink）
- 复制节点
- 删除节点
- 禁用/启用节点
- 查看日志

### 17.4 简单节点（Simple Node）

**用途**：简化的节点渲染组件，用于特定场景（如预览模式）。

**状态支持**：
- 运行状态：Running/Succeeded/Failed/Exception
- 等待运行状态（_waitingRun）
- 候选节点状态（_isCandidate）
- 捆绑状态（_isBundled）
- 进入状态（_isEntering）

### 17.5 节点重试配置（Retry）

**位置**：节点配置面板 → 高级设置。

**字段**：
1. **启用重试**：开关
2. **最大重试次数**：1-10 次（滑块 + 数字输入）
3. **重试间隔**：100-5000ms（滑块 + 数字输入）

**适用节点**：LLM、HTTP Request、Tool、Agent、Knowledge Retrieval

### 17.6 节点错误处理（Error Handle）

**位置**：节点配置面板 → 高级设置。

**错误处理类型**：
1. **无处理（none）**：错误时工作流失败
2. **失败分支（failBranch）**：错误时走失败分支（新增输出端口）
3. **默认值（defaultValue）**：错误时返回预设默认值

**失败分支卡片**：
- 显示失败分支输出端口
- 支持连接到其他节点

**默认值表单**：
- 根据节点输出类型动态生成表单
- 支持 string/number/object/array 类型

### 17.7 发布前检查清单（Checklist）

**触发时机**：点击"发布"按钮时弹出。

**检查项分类**：
1. **节点检查**（ChecklistNodeGroup）
   - 未配置的节点
   - 缺少连接的节点
   - 缺少输入变量的节点
2. **插件检查**（ChecklistPluginGroup）
   - 缺失的插件依赖
   - 插件版本不兼容

**交互**：
- 显示检查项计数（如 "3 个问题"）
- 支持"转到"功能跳转到问题节点
- 修复所有问题后才能发布

### 17.8 插件依赖管理（Plugin Dependency）

**功能**：管理工作流依赖的插件。

**特性**：
- 从 DSL 自动加载插件依赖
- 插件安装弹窗（InstallBundle）
- 依赖列表管理
- 版本兼容性检查

### 17.9 工作流预览模式（Workflow Preview）

**用途**：只读模式，用于查看已发布的工作流。

**三个 Tab**：
1. **INPUT Tab**：显示输入变量表单
2. **TRACING Tab**：显示执行追踪
3. **RESULT Tab**：显示最终输出

**特性**：
- 支持人工输入表单显示（humanInputFilledFormDataList）
- 支持输入面板展开/收起
- 支持面板宽度调整
- 禁用所有编辑操作

### 17.10 全局变量面板（Global Variable Panel）

**位置**：Header → 全局变量按钮。

**系统全局变量**：

| 变量名 | 类型 | 说明 | 适用模式 |
|--------|------|------|---------|
| `sys.query` | string | 用户输入 | Chatflow |
| `sys.files` | array[file] | 上传文件 | 全部 |
| `sys.conversation_id` | string | 对话 ID | Chatflow |
| `sys.user_id` | string | 用户 ID | 全部 |
| `sys.dialog_count` | number | 对话轮次 | Chatflow |
| `sys.timestamp` | number | 时间戳 | Workflow |

### 17.11 对话变量面板（Chat Variable Panel）

**位置**：Header → 对话变量按钮（仅 Chatflow）。

**功能**：
- 管理对话级别的自定义变量
- 支持变量增删改操作
- 支持变量使用影响分析（findUsedVarNodes）
- 支持变量重命名时自动更新节点引用
- 支持删除前确认（RemoveEffectVarConfirm）
- 支持与协作系统同步（WebSocket）

**变量影响分析**：
- 显示哪些节点引用了该变量
- 删除变量前警告影响范围

### 17.12 评论面板（Comments Panel）

**位置**：Header → 评论按钮。

**功能**：
- 显示工作流上的所有评论
- 支持评论过滤：
  - 已解决/未解决
  - 我的/全部
- 支持评论解决标记
- 支持评论参与者头像显示
- 支持回复计数显示
- 支持活跃评论高亮

### 17.13 对话记录面板（Chat Record Panel）

**位置**：Chatflow 调试预览面板 → 对话记录 Tab。

**功能**：

### 17.14 Agent 执行日志（Agent Log）
