## 4.1.2 状态机（S1-STM）——本切片风险最高处，矩阵驱动全覆盖

| ID | 优先级 | 用例 | 步骤 → 预期 | 层/方式 | 节点 |
|----|-------|------|------------|--------|------|
| S1-STM-01 | P0 | 合法流转 | draft→pending（提交审核）、pending→published（审核通过）、pending→draft（驳回）、published→offline（下线）、offline→published（重新上线，按设计需再审）→ 全部成功且留审计记录 | L2/Vitest | 4.1.2 |
| S1-STM-02 | P0 | 非法流转全拒绝 | 4 状态 × 4 目标全矩阵中所有非法组合（如 draft→published 跳审核、offline→pending）→ 服务端 409/422 拒绝，状态不变 | L2/Vitest（参数化） | 4.1.2 |
| S1-STM-03 | P0 | 越权流转 | Developer 尝试审核通过自己的 Agent → 403（提交人与审核人分离）；User 尝试任何流转 → 403 | L3/Vitest | 4.1.2 |
| S1-STM-04 | P1 | 前端状态同步 | 每次流转后 UI 状态徽章、可用操作按钮同步变化（draft 显示"提交审核"，published 显示"下线"） | L4/Playwright | 4.1.2 |

## 4.1.3 审核流程（S1-REV）｜PRD 2.7

| ID | 优先级 | 用例 | 步骤 → 预期 | 层/方式 | 节点 |

| S1-REV-04 | P2 | 重复提交防护 | 已在 pending 的 Agent 再次提交 → 幂等（不产生第二条审核单） | L3/Vitest | 4.1.3 |

## 4.1.4 真实对话（S1-CHAT）｜数字员工接通大模型
