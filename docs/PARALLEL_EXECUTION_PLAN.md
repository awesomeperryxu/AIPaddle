## 5. 🔴 测试门（全类型六层，测完才进下一步）

测试分层见 `docs/TESTING.md`。**每个任务/切片/阶段都有对应测试门，不绿不过。**

| 时机 | 必跑测试 | 谁 |
|---|---|---|
| **写码时** | L1 静态（TS 实时报错）| 开发道 |
| **每个任务完成**（关 Issue 前）| L1 `pnpm check` + L2 单元（涉逻辑）+ L3 集成/API（涉接口）+ L6① 用户手动验收 | 开发道 + 你 |
| **每次 push** | CI 自动跑 L1+L2+L3 | CI |
| 🔴 **每个切片完成**（进下一切片前）| L4 E2E（该切片 Playwright 集 S1~S5）+ L5 专项：**隔离(P0)** 必跑；知识库切片加 **L5 RAG 金标准评测**（命中率≥80% 引用≥90%）；性能/安全按需 | 开发道 |
| **每个阶段完成** | L6② 真实用户测试 | 你 |

**硬性要求落地**：
- 每个 Issue 必须走完 `.github/ISSUE_TEMPLATE/task.md` 的 9 步，**测试项全绿才能 close**。
- 一个切片的 **L4+L5 未全绿，该道不得开下一切片的任务**。
- 自动修复最多 3 轮（只修码不改测试），3 轮仍红打 `status:blocked` 交你裁决。

## 6. 协调铁律

1. **每道独立 git worktree**（`git worktree add ../wt-<lane> -b feat/<lane>-xxx origin/main`）；node_modules 各装（不能软链，turbopack 拒绝）。
2. **小 PR、勤合并**：一个任务一个 PR，CI 绿 + S0-ISO 绿即合。集成是唯一串行点，越勤越不痛。
3. **E 道是合并/集成 owner**：负责保持 main 绿、S0-ISO 绿、路由/外壳/permissions 整合冲突的裁决。
4. **A 道是 schema owner**：见 X3。
5. **完成即三处同步**（ROADMAP + 进度 Excel + 关 GitHub Issue）。
6. DB 直连方式（跑迁移/RLS 排查）见 memory：从 Crab 走 IPv4 Session Pooler（ap-southeast-2）。
