# ADR-003：大模型接入方案

- 状态：**已采纳**（2026-07-18，Perry 拍板）
- 对应任务：ROADMAP 2.7（并闭环 2.6 轮子清单的"模型网关"项）
- 关联：ADR-001（Supabase/pgvector）、数据模型 C5（向量 1536 维）

## 决策

1. **默认对话模型：Kimi 2.5（Moonshot API）**。Moonshot API 兼容 OpenAI SDK 格式，服务端统一经 OpenAI 兼容客户端调用（`baseURL` 指向 Moonshot）。
2. **功能上可自主调配**：每个 Agent 可独立指定模型——`agents.config.model` 字段（jsonb，数据模型已预留），未指定则回落到租户默认 → 平台默认（Kimi 2.5）。切换模型/供应商不改代码，只改配置。
3. **API Key 管理**：所有模型 Key 仅存服务器环境变量（`MOONSHOT_API_KEY` 等），绝不入库明文、不进前端、不进 git（延续 ADR-001 约定；切片 1 的 S1-CHAT-03 用例专门验证）。
4. **模型网关**：首版**不引入** LiteLLM 等网关——单一供应商直连即可，少一层运维。触发引入条件：接入第 3 家供应商、或需要成本/质量路由与统一计量时（PRD 2.9.2 的多模型 Fallback/路由属阶段 6）。
5. **用量记录**：每次调用写 `call_logs`（tokens_in/out、model、latency、success），为 Dashboard 统计与将来计费（阶段 6）提供数据底座。

## 待办（不阻塞当前批次）

- **嵌入模型选型（切片 2 前必须锁定）**：Moonshot 不提供 embedding API。约束：输出 1536 维（与 chunks.embedding 一致）或迁移时同步改列。候选：
  - OpenAI `text-embedding-3-small`（1536 维，生态标准）
  - 硅基流动/智谱等国产兼容服务（注意多为 1024/2048 维，选用则改 C5 决策并重建列）
- RAG 拒答与引用的提示词模板（切片 2 开发时定稿，金标准评测把关）。

## 被否决的备选

| 方案 | 否决原因 |
|------|---------|
| 首版即上 LiteLLM 网关 | 单供应商阶段纯增复杂度；触发条件明确后再上 |
| 模型 Key 存数据库供租户自配 | PRD 2.9.2 的租户级模型管理属阶段 6，首版平台统一管 Key |

## 复审条件

接入第 3 家模型供应商；出现租户自带 Key（BYOK）需求；Kimi 2.5 成本或质量不达预期。
