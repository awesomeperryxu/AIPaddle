# ADR-006：办公文件处理能力（本地文件读写三通道）

- 状态：**已采纳**（2026-07-18，Perry 拍板：三通道全选 + 企业微信微盘，定位为平台高优先级基础能力）
- 关联：ADR-004（MCP 治理）、ADR-005（Copilot 工具注册表）、PRD v1.08 §2.12

## 决策

用户经 AI Copilot 处理办公文件（总结/改写/翻译/生成 Word/Excel/PPT/PDF），按三通道分期落地：

| 通道 | 能力 | 排期 | 说明 |
|------|------|------|------|
| ① 上传-处理-下载 | 拖入文件 → AI 处理 → 下载结果 | **切片 2（任务 4.2.4）** | 全浏览器兼容；解析复用切片 2 知识库管线；生成用平台内置库（docx/xlsx/pdf 固定代码，非 AI 写码）→ **无需沙盒** |
| ② 企业云盘 MCP | 飞书云文档、金山 WPS、**企业微信微盘**、OneDrive | **切片 3（任务 4.3.4）** | 走 ADR-004 治理原路：注册 MCP Server → 封装 Skill → 权限/白名单/审计全复用，零新架构。企微微盘无现成 MCP，需按企微开放 API 自建 Server（enterprise 类型） |
| ③ 浏览器授权直读写 | File System Access API，授权文件夹后直接读写本地文件 | 二期（切片 6，核心闭环后） | 仅 Chromium 内核（Chrome/Edge）；每会话需用户重新授权（浏览器安全模型，不可绕过）；写回前展示 diff 供确认 |

## 安全模型（延续 ADR-005，不引入沙盒）

- AI 只调用注册表中的固定工具（`parse_document / summarize / rewrite / generate_docx / generate_xlsx / generate_pdf / drive_read / drive_write`），工具是平台代码，AI 不产出可执行代码；
- 通道③ 的文件写入发生在**用户浏览器内、经用户授权的目录**，服务端不触达用户磁盘；写回操作必须先向用户展示变更预览（diff/新旧对照）并经确认；
- 云盘写操作按 ADR-004 切面封装（如「微盘只读」低风险全员 /「微盘写入」中风险指定角色）；
- 上传文件存 Supabase Storage（org 隔离），处理记录写 audit_logs，模型调用写 call_logs。

## 工程落点

- Copilot 工具注册表（ADR-005 附录1）新增 file 域工具条目；循环零改动；
- 文档解析与 4.2.2 知识库管线共用同一套解析器（一次开发两处使用）；
- 文件生成库选型随任务 4.2.4 定（docx/xlsx/pptx 生成为成熟开源库，属实现细节）。

## 复审条件

通道③ 浏览器兼容性诉求扩大（考虑桌面客户端）；企微/飞书 API 政策变化；文件处理量大到需要独立处理队列（引入 worker 时豁免单体）。
